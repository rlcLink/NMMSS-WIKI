This function produces a formatted report of all transactions whose shipper and receiver data, including adjustments, are not compatible.

**Use Case(s)**

- UC2.1 - The Nuclear Material Analyst executes a Transaction Processing report.

**System Requirement(s)**

- ITSCR15 - The NMMSS shall provide a consistent user interface.

- RPTFMT1 - The NMMSS shall use a standard report format for all reports.

- RPTFMT2 - The NMMSS shall begin each report with a standard cover page.

- RPTFMT3 - The NMMSS shall, for each internal page, follow a standard header and footer format.

- RPTFMT4 - The NMMSS shall end each report with a standard end of report page.

- RPTFMT5 - The NMMSS shall indicate blank report no data found in the body of NMMSS reports where a report finds no data for the specified input parameters.

- RPTFMT6 - No report produced by the NMMSS shall alter the data from which it was produced.

- RPTFMT7 - The NMMSS shall not allow information to be retrieved from the database for a report, for a query, or for a transaction while any of the requested information is in the process of being updated.

- RPTFMT10 - The NMMSS shall have the ability to produce a page that lists query parameters used to generate report

- SEC11 - The NMMSS shall, for each page of any NMMSS report, meet the security requirements for the marking of classified and unclassified documents.

- TRRPT10 - The NMMSS shall be able to produce a transaction work list for ADP control report (TJ-003B) that lists all transactions whose shipper and receiver data, including adjustments, are not compatible. This report is used by NMMSS analysts to contact the facilities whose data is in question to resolve the discrepancy.
